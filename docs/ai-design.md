# AI 电脑玩家设计（当前版）

本文件是电脑玩家（AI 自动机）的**当前唯一权威文档**，覆盖控制器接口、价值模型、目标系统、回合规划、自博弈验证和后续路线。旧的接口契约文档与成本收益草稿已经合并进本文，不再单独维护。

- **控制器接口**：浏览器内由 `randomizer/app/ai-controller.js` 识别电脑玩家、推进 pending 子决策、执行顶层行动，并暴露批跑 / A/B / 调参入口；规则估值层集中在 `randomizer/game/ai/**`。
- **大脑层**：价值模型、目标系统、回合规划器，以及它们如何把“当前可行动内容的实时成本/收益”和“长线达成目标的动态收益”统一成一条决策链路。
- **核心规则**：
  1. **收入价值要扣掉丢牌成本**：一次收入提升会弃 1 张手牌，价值 = 资源价值×剩余可享受次数 − 被弃牌价值。
  2. **价值以“动态实时框架单位”计算**：每次决策实时枚举可行动内容并算 `净收益 = 收益 − 成本`；长线收益（完成扇区 / 凑 6 数据分析 / 登陆等）按**当前可达性**动态加权，且因对手行动而实时变更。
  3. **终局板块标记后围绕高价值终局计分行动**：标记 25/50/70 后，凡能推进所标记 a/b/c/d 公式的行动，在原收益上叠加对应位次倍率的终局边际分。
  4. **默认人机入口按 1 人类 + 3 电脑运行**：普通浏览器入口保留白色人类玩家，其余 3 个活跃席位由电脑行动；批跑和 A/B 入口未显式传 `activePlayerCount` 时也按 4 人局重置。

> 机制、状态模型或接口变化时，请同步本文与 `AGENTS.md`（见 `AGENTS.md` 约定）。

## 1. 当前版解决的问题

当前 AI 是“单步贪心 + 被动 routeDemand”，主要短板（基于 `game/ai/policy.js`、`evaluator.js` 和 `app.js` 的 `scoreAi*` 链）：

| 短板 | 现状 | 当前对策 |
|------|------|---------|
| 价值口径分散、常数手写 | 估值散落在 `evaluator.RESOURCE_VALUES` 与十几个 `scoreAi*` | 收口为单一 **行动收益/成本图谱**（§3、§4） |
| 收入估值偏高 | `getIncomeValue` 只乘剩余轮次，未扣丢牌成本 | **收入净值**公式（§3.3） |
| 只有单步边际分 | 取候选最大分，不规划多步兑现链 | **回合浅前瞻**（§6） |
| 无显式目标 | `routeDemand` 被动反推偏好 | **目标系统**（§5）：首痕迹三策略 / 第一轮 25 分 / 长线达成 |
| 终局板块利用不足 | 仅在达阈值时处理 `final-score-mark` | 标记后**全程叠加终局边际价值**（§4.4） |
| 行动覆盖窄 | `canAiResolvePlayCardEffects` 仍有未覆盖类型，外星人使用链过去常阻塞自动机 | 按目标价值排序逐步收口（§7） |

## 2. 总体架构与接口契约

四层，自下而上，每层纯函数化、可单测、可日志化：

```
┌─────────────────────────────────────────────────────────┐
│ L4 目标系统 Goals            game/ai/goals.js             │ 开局选定 + 轮内复评
│   首痕迹(黄/粉/蓝) / 第一轮25分 / 完成扇区 / 6数据分析 / 登陆 │
├─────────────────────────────────────────────────────────┤
│ L3 回合规划器 Planner        game/ai/planner.js           │ 浅前瞻(深2~3, beam)
│   枚举“快速→主→后置快速→end/pass”链，选整条链净值最优      │
├─────────────────────────────────────────────────────────┤
│ L2 行动收益图谱 ActionGraph  game/ai/action-graph.js      │ 动态实时单位
│   每个候选 -> {gain, cost, net, prereqChain, breakdown}   │
├─────────────────────────────────────────────────────────┤
│ L1 价值模型 Valuation        game/ai/valuation.js         │ 单一估值口径
│   资源折算 / 收入净值 / 终局边际 / 状态估值                 │
└─────────────────────────────────────────────────────────┘
        ▲ 接入 app/ai-controller.js 的电脑玩家驱动与子决策收口
```

数据流（一次决策）：

```
GameState 快照
  → L4 给出当前激活目标与权重（含对手影响后的可达性）
  → L2 对每个 legal action 实时算 net（含终局边际、目标加权）
  → policy / L3 优先使用 L2 的 `actionGraph.net` 做选择；旧 `candidate.score` 只作为 fallback 与 tie-breaker
  → 经 `app/ai-controller.js` 执行，子决策再回到 L2/L4 估值
  → battle-analytics 记录 breakdown，供调参与路径挖掘
```

设计约束：估值只读、克隆 `createGameRecoverySnapshot()` 做“试一步→回退”；`irreversible` 步骤不进搜索回退，遇到用启发式即时决策。

### 2.1 当前接口契约

- 电脑玩家判定由 `isAiAutoBattlePlayer(playerId)` 统一处理；默认人机局保留白色人类玩家，其余活跃玩家由 `configureDefaultAiOpponent()` 配置为电脑。
- `createAiControlSnapshot()` / `restoreAiControlSnapshot()` 只保存和恢复可持久化 AI 控制配置：是否启用、电脑席位、步进参数和策略权重；不恢复 `running`、已排队定时器、进行中的自动步骤或旧的阻塞暂停。旧存档缺失 AI 控制配置，或快照中的电脑席位无法解析时，按默认人机入口重建电脑席位；显式 `enabled:false` 的快照仍按全手动恢复。
- 批跑 / A/B / 调参入口走 `configureAiAutoBattle()`、`runAiAutomationStep()`、`runAiAutoBattleBatch()`、`runAiStrategyABTest()`、`runAiStrategyTuningCycle()`；未显式传 `activePlayerCount` 时按 4 人局重置。
- `runAiAutomationStep()` 是唯一推进器，先收口外星人使用、外星人痕迹和半人马就绪机会，再按“初始选择 / 弃牌 / PASS 预留 / 终局标记 / 公共牌选择 / 科技放置 / 扫描 / 打牌 / 移动支付 / 登陆 / 数据放置 / 共用扫描弹窗 / 效果链 / 顶层行动”的顺序推进其余 pending 状态。
- 顶层行动候选仍由现有规则入口判断可用性，策略层优先读取 `actionGraph.net`，旧 `candidate.score` 只作为 fallback 与 tie-breaker。
- 子决策以 `runAi*Decision()` 族函数处理；每个 AI pending 分支必须返回 `progressed`、`skipped` 或明确 `blocked`，不能把自动批跑永久停在需要人工点击的状态。
- 任何跨弹窗延迟结算的 pending 都必须带 `playerId/playerColor` 或可解析 `effect`；AI 分发和确认 handler 均按 `pending owner -> effect owner -> current player` 解析执行玩家。若 pending owner 是人类玩家，AI 必须返回 `blocked` 等待人工处理；不能因为当前可见玩家是人类而让人类控制 AI pending，也不能因为当前可见玩家是 AI 而代处理人类 pending。
- 共用 overlay 的 rare 效果也要有 AI 收口路径。`scanTargetOverlay` 除普通扫描外还承载移除标记、手牌角标奖励、任意弃牌收入、支付信用、重复角标、移除环绕放探测器、任务回手、探测器扇区扫描和探测器位置奖励；这些 pending 即使仍不作为 AI 主动打牌候选，也必须在被强制或触发进入时能自动选择、跳过或确认，不能停在弹窗上。
- 1 类任务卡触发只在当前事件窗口内可选；AI 若因火箭上限、资源不足或无合法目标无法发动某个触发，应取消本次触发选择并等待下次同类事件，不得消耗触发槽，也不得把自动批跑阻塞在不可执行触发上。
- AI 不直接改规则状态；真正执行仍走人类同路径的 `runAction()`、`beginScanAction()`、`beginPlayCardSelection()`、`researchTechForCurrentPlayer()`、`passForCurrentPlayer()` 等入口。
- 随机性通过现有 seeded 批跑入口和可注入 `random` 的规则函数复现；不能为了 AI 策略在估值阶段消耗真实随机结果。

## 3. L1 价值模型（单一估值口径）

集中在 `game/ai/valuation.js`，把基础价值、行动成本和路线收益统一编码，替换分散的常量。

### 3.1 资源折算单位

| 资源 | 价值（分） | 说明 |
|------|-----------|------|
| 1 分 | 1 | 基础分 |
| 1 信用点 / 1 能量 / 1 精选 | 3 | 资源与精选基础折算 |
| 1 移动 / 1 数据 / 1.5 宣传 | 1.5 | 移动按真实支付成本重估 |
| 1 信号（含数据） | 3 | 扫描和数据链基础收益 |
| 普通手牌 | ≈ 3（≈ 1 盲抽） | 视具体任务、终局牌和角标动态调整 |
| 外星人卡 | ≈ 4 | 多数外星人卡高于普通牌 |

> 口径统一后，调参从“改十几个魔法数字”变为“改一张表”，且每个候选可输出 `breakdown` 便于解释。

### 3.2 状态估值

`evaluate(state, playerId)` = 终局/实时分（复用 `endGameScoring.computePlayerFinalScore`，`end-game-scoring.js`）+ 资源折算 + **收入净值** + 手牌价值 + 保留牌价值 + **已激活目标的期望达成值**。

### 3.3 收入净值（★ 核心修正）

机制（`mechanics-reference.md`）：一次“收入提升”要**弃 1 张手牌**，按该牌收入角标提升收入档；提升后的收入在之后每个收入结算点都生效。

旧估值问题：`evaluator.getIncomeValue` 仅 `资源价值 × 剩余倍率`，**未扣丢牌成本**，导致高估收入类行动。

当前公式（评估“获得 +ΔR 收入”这一行为的净值）：

```
incomeNet(ΔR) = resourceValue(ΔR) × remainingIncomeTimes(round) − discardedCardValue
```

- `resourceValue(ΔR)`：本次提升的收入资源折算（常规基准如 +1 信用 = 3 分/次；前 2 轮 AI 估值会把信用点/能量抬到约 6，并抬高手牌上限/精选收入；收入选择顺序倾向能量 > 信用点 > 手牌，以反映早期资源比即时分更关键并避免末期能量或手牌断档）。
- `remainingIncomeTimes(round)`：该收入还能享受几次。第一轮 4 次（获得当次 + 第 2/3/4 轮各一次），逐轮递减。注意 PASS 轮的收入结算细节（第 4 轮 PASS 不获得收入，见 `mechanics-reference.md`）要并入次数计算。
- `discardedCardValue`：被弃手牌价值。估值时取“AI 实际会弃的那张”的价值（通常是当前手里最低价值的牌，但不低于普通牌基准 ≈ 3）。这是旧估值缺失的关键减项。

推论：收入在**早轮**且**手牌富余**时才划算；晚轮或手牌紧张（接近上限/缺关键牌）时收入净值可能为负，AI 应少做。

### 3.4 行动成本要算全链

环绕/登陆的成本 = **发射 + 移动 + 行动本身**。普通发射支付成本按 2 信用点；`skipCost` 发射为 0；带 `cost` 的卡牌/科技发射按指定资源。普通移动每点移动力在 1 能量和 1 张可支付移动的手牌之间取较低估值，小行星带离开仍需要 2 点移动力。移动步数过高的星球，收益往往补不回成本。

AI 对移动候选必须拆分真实 `paymentCost` 与 `pathPenalty`：卡牌/科技给的免费移动只保留路径质量判断，不额外扣能量或手牌；小行星额外移动、远离目标、无后续主行动的绕路和向内掉头只作为风险/机会成本写入 `pathPenalty`，避免飞船卡在小行星带或为了微弱方向偏置移动过远。无紫科的主行动扫描成本 ≈ 1 信用 + 2 能量 = 9 价值、裸收益仅 2 信号，需依赖 2 号数据位、赢扇区、紫科加成才转正。L2 图谱必须把前置链成本并入候选 `cost`。

## 4. L2 行动收益图谱（动态实时单位）

`game/ai/action-graph.js`：把“当前可行动内容”实时映射为统一候选结构。**每次决策都重算**，因为对手行动会改变成本/收益（主星环绕/登陆的奖励序号变化、卫星登陆位被占、扇区被别人完成、痕迹被抢）。

### 4.1 候选结构

```
ActionCandidate = {
  id,                 // launch | orbit | land | analyze | scan | playCard | researchTech | move | industry | quickTrade | pass | end-turn
  gain,               // 直接收益（资源/分/信号，按 §3.1 折算）
  cost,               // 全链成本（含前置发射+移动，按 §3.4）
  prereqChain,        // 兑现所需的前置步骤（用于 L3 浅前瞻）
  finalMarginal,      // 终局板块边际价值（§4.4），无标记时为 0
  goalBonus,          // 激活目标加权（§5）
  feasibility,        // 0~1，可达性（受对手影响，§4.3）
  net,                // = (gain + finalMarginal + goalBonus) × feasibility − cost
  breakdown,          // 各项明细，进日志
}
```

`net` 是决策主依据，`breakdown` 是可解释性与调参依据。

### 4.2 实时枚举

来源沿用现有判断（`actions.canExecute`、`scanEffects.canExecuteScan`、`data.canAnalyzeData`、`quickTrades.canExecuteTrade`、`canStartMainAction()` 等）。图谱层只读，不改状态。

### 4.3 长线目标的动态收益（★ 受对手影响）

“完成扇区 / 凑 6 数据分析 / 登陆某星球”这类**跨多回合**的收益，不能当成固定常数，要按**当前可达性**动态折算，并随对手行动实时更新：

- **完成扇区**：价值 = 完成时收益（含 b2 终局、扫描奖励）× 进度系数。当某扇区被对手抢先完成或填满时，剩余收益下降甚至清零 → `feasibility` 实时回落。
- **凑 6 数据分析**：价值随“距离 6 还差几个数据”递增（越接近越高）；若数据来源（星云/扫描机会）被对手消耗，达成概率下降。
- **登陆 / 环绕特定星球**：普通主星环绕/登陆可无限累计，超过参考图贴图数后只计数和结算奖励；AI 仍要按当前下一枚标记序号重算奖励价值。卫星登陆和冥王星等特殊目标仍有唯一性或卡牌状态限制，被占或不可用时对应目标 `feasibility → 0`。
- **首痕迹**：见 §5；被他人抢占后对应目标价值清零。

实现要点：长线目标在 L4 维护“期望达成值 + 当前进度 + 可达性”，L2 把“推进该目标的行动”按**边际进度**折算进 `goalBonus`/`finalMarginal`，而非一次性给满。

### 4.4 终局板块边际价值（★ 标记 25/50/70 后）

终局板块 a/b/c/d 在玩家分数达 **25 / 50 / 70** 时各标记一次，位次（1/2/3 及之后）决定倍率（`assets/final/final_detail.md`）。标记后，凡推进对应公式的行动都要在原收益上叠加 `finalMarginal`：

```
finalMarginal(action) = Σ_overMarkedTiles  positionMultiplier(tile) × marginalUnitDelta(tile, action)
```

各公式的“边际单位”与受影响行动：

| 板块 | 公式要义 | 推进该公式的行动 | 边际注意 |
|------|----------|------------------|----------|
| a1 | 收入中信用/能量较多者 | 提升信用或能量收入 | 只对“较多的那侧”加分 |
| a2 | 信用/能量/盲抽收入最小值 | 提升当前最小的那项收入 | min 型：只对短板加分 |
| b1 | 三色痕迹最小值 | 放置当前最少颜色的痕迹 | min 型：补短板 |
| b2 | min(环绕+登陆, 完成扇区) | 环绕 / 登陆 / 完成扇区 | min 型：补较少的一侧 |
| c1 | 完成任务数（1+2 型） | 完成任务牌 | 线性 |
| c2 | (完成任务+终局牌)/2 向下取整 | 完成任务 / 打终局牌 | 每 2 个才 +1 单位 |
| d1 | 三色科技最小值 | 研究当前最少颜色的科技 | min 型：补短板 |
| d2 | 科技总数/2 向下取整 | 研究任意科技 | 每 2 个才 +1 单位 |

关键：**min 型与“除 2 取整”型公式是非线性的**——边际分取决于当前计数，必须用“做完该行动后公式值 − 当前公式值”实时计算，不能用平均常数。位次倍率越高（1 号位）的板块，对应行动越要多做。

## 5. L4 目标系统

`game/ai/goals.js`：把开局意图固化为显式目标对象，**开局选定 1~2 个主目标，轮内复评可达性**，避免每步重算导致摇摆。

### 5.1 目标接口

```
Goal = {
  id, priority,
  progress(state),            // 当前进度 0~1
  feasibility(state),         // 当前可达性 0~1（受对手影响实时变化）
  expectedValue(state),       // 达成期望分（× feasibility）
  bonusFor(candidate, state), // 该候选对本目标的边际贡献 -> goalBonus
}
```

### 5.2 内置目标

- `FIRST_ROUND_SCORE_25`：跟踪实时分与 25 的差距，对“能直接加分/凑分链”的候选加 `goalBonus`；与首痕迹目标协同（首痕迹本身通常约 5 分 + 宣传 + 外星人卡或物种揭示收益）。
- `GRAB_TRACE_YELLOW`（抢登陆）：目标星球登陆链高优先；黄痕迹竞争激烈，`feasibility` 随对手逼近而下降。
- `GRAB_TRACE_PINK`（赢扇区扫描）：目标“本轮赢一个扇区扫描”，把扫描/数据投放绑定到该扇区。
- `GRAB_TRACE_BLUE`（6 数据分析）：目标“尽快凑 6 数据并分析”，把数据获取与分析行动连成引擎。
- `FINAL_TILE_FOCUS`：达 25/50/70 标记后激活，把 §4.4 的 `finalMarginal` 提升为高优先目标导向。

### 5.3 开局规划

读公司牌 / 起始牌 / 初始手牌，评估：哪条首痕迹可行（黄/粉/蓝）、第一轮能否摸到 25 分、能否实现“4 数据收入 / 6 数据分析”。选定主目标后写入 L4，供后续每回合候选评分叠加。

## 6. L3 回合规划器（浅前瞻）

`game/ai/planner.js`：用 `createGameRecoverySnapshot()` 做“试一步→回退”，深度 2~3、受限宽度的 beam search：

- 枚举一回合内“快速行动 → 主行动 → 后置快速行动 → end-turn/PASS”的组合链，用 L1 `evaluate` 估每条链的终局状态价值，选**整条链净值最优**而非单步最优。当前主链已纳入 `analyze`，快速链已纳入 `placeData`，避免 AI 拿到数据后不投放、不分析。
- 重点解决“发射 → 移动 → 登陆/环绕”这类需多步兑现的高收益链（当前打不出高分的主因）。
- `irreversible`（翻外星人牌、随机抽牌）不进回退，用启发式即时决策。
- PASS 作为候选显式权衡：轮序、收入净值（§3.3）、剩余主行动机会成本。

## 7. 行动覆盖扩展

逐步消化 `canAiResolvePlayCardEffects` 的 `unsupportedTypes` 与外星人卡，**按对当前激活目标的价值排序**优先收口（先扫描/登陆/数据相关）。每收口一类在 `game/ai/ai.test.js` 加回归。

已完成的第一批覆盖：

- 顶层行动选择已优先使用 `actionGraph.net`；运行时会记录 `gain/cost/finalMarginal/goalBonus/feasibility/net`，收入估值调用 `valuation.getIncomeNetValue()`，已标记终局公式会实时叠加边际分。
- 前两轮的信用点/能量估值会按早期资源溢价进入资源、收入、真实移动支付、登陆和分析成本；移动候选显式拆分 `paymentCost/pathPenalty`，卡牌免费移动不扣能量/手牌，高移动点、小行星阻力、远离目标和负收益免费移动会被压低或跳过。
- 移动路径会额外评估旋转等待窗口和小行星陷阱：无论当前路线目标是否是行星，AI 都会检查移动后到最近可操作星球是否仍超过最优距离；若三次旋转可能自然缩短距离（火星约 1、木星/土星约 2、天王星/海王星约 3），无后续兑现的移动会增加等待惩罚。没有橙2时，多火箭进入小行星且仍离最近可操作星球较远会被强烈压低，若只是把小行星本身当路线目标且不能继续同一火箭移动/登陆兑现，也会按真实离场阻力追加风险成本；橙2科技会按当前小行星、多火箭、寰宇免费移动压力和远离可操作星球的路线压力获得额外价值。无橙2时，寰宇/寰宇超动力移动停到小行星且没有立即兑现会额外叠加组合风险，尤其压低两枚火箭都进小行星的路线；若寰宇 1x 没有任何正收益免费移动，则不再启动后空转。
- 早期信用点、能量和手牌价值进一步抬高；直接得分会按 25/50 节奏获得更高的早期 pace bonus，收入选择会对能量/手牌收入短板补偿、对信用点明显过剩降权；发射、环绕、登陆、扫描会额外惩罚把信用点/能量压到低储备的动作，避免前两轮为短期得分透支后续转换能力，也避免只堆信用点导致后期没有手牌/能量转换。
- 一次性资源、星云数据、蓝科附加数据槽和科技 bonus 会额外估算“是否补齐下一步行动成本”：信用点打开打牌/扫描，能量打开扫描、分析、登陆/环绕兑现，宣传打开精选牌，数据推动 6 数据分析。第 5/6 个计算机数据在第 2/3 轮会按临近分析窗口额外加权；蓝色科技研究也会按当前数据池、计算机进度、蓝痕迹/分析需求获得路线加成，能量 bonus 会更重视中局分析/扫描/登陆续航，服务“数据 -> 分析 -> 得分/终局板”的中期引擎。
- 当计算机第 6 位已经放置且分析可执行时，AI 会把分析视为主行动现金化窗口：第 3 轮前后若已有 5-6 数据、蓝痕迹/分析需求或仍缺 50/70 终局标记，分析会获得额外机会成本加分，避免先打牌后主行动窗口关闭，只能把已成熟的数据链推迟一整回合。
- 第 1 轮若分数仍低于 25 且能量、手牌都为 0，额外发射会被压低，优先让 AI 用 PASS 或收入刷新行动经济，避免空资源继续铺火箭导致后续断档。
- 收入弃牌在收入终局公式或路线能量短缺场景下使用玩家当前资源、收入短板和路线需求动态排序，其余场景回退到纯策略层的固定收入顺序。
- 收入结构策略进一步收紧：第 1/2 轮默认信用点收入优先，能量收入只作为路线续航短板补充；盲抽收入到 2 后继续提升会被显式降权，除非 A2 收入终局公式或明显手牌断档能抵消该惩罚。避免信用点收入不足时把额外手牌变成低效快速奖励或 2 牌换 1 资源。
- 高分但高成本的直接换分行动按轮次分阶段估值：半人马 3 数据 15 分、天王星/海王星卫星等在第 1/2 轮会扣除资源滚动损失，第 3 轮只在能跨 50/70 或资源足够时放松，第 4 轮转为抢高分兑现。半人马揭示后，若手里有半人马牌，能量收入会额外按“先打牌建立 +15 阈值，再兑现条件奖励”的路线估值；半人马 3 数据 15 分痕迹在第 1/2 轮和蓝色科技较多时继续降权，除非能直接跨终局分数门槛。天王星/海王星卫星作为后续路线目标时还会检查前序对手是否已有橙4且已到达对应星球，降低“白花移动/能量后被抢位”的路线价值；若第 4 轮已在 3 步内或可立即登陆，AI 会把 25/26 分卫星视为明确现金化目标，缺 1-2 能量时也按临门补资源目标保留额外溢价，并对确实缩短到 3 步内的移动给路线接近加权，反向提高橙4研究与移动路线权重。
- 终局轮临近结束且只差少量分数跨 50 时，AI 会把 `2信用点→1能量` 或 `2张牌→1能量` 作为分析救急候选，但必须满足分析后的蓝痕迹/方舟奖励能实际跨 50，或已经 49 分且数据池/计算机数据足以完成 6 数据分析；避免 47 分只因已标记收入终局公式就提前交易，导致能量被移动等行动消耗却没有拿到第二个标记。若已换出能量且数据池足够重装 6 数据，45-49 分的一标记玩家会提高数据放置优先级，确保能再次分析兑现蓝痕迹。
- 普通环绕和登陆都会移除火箭，AI 不再把同一枚火箭的“环绕后登陆”累加为一条终局兑现链；终局低于 50 且缺第 2 个标记时，0 直接分的环绕会额外扣分，避免移除唯一可登陆火箭后失去跨 50 的机会。
- 终局低于 50 且只有 1 个终局标记时，研究科技会显式计算首拿 +2 分、`bonus_3f`、`techTypeCountScore`、`resourceValueScore` 和 `repeatBonus` 等直接得分；若这些直接分仍不足以跨 50，D1/D2 终局公式目标加成会被压低，避免 AI 在第二个标记前继续追科技终局边际而错过可兑现分数。
- 直接得分跨过 50/70 的现金化价值已收口到 `valuation.estimateFinalMarkCashoutValue()` 并进入 `action-graph` 统一候选 `net`；终局缺 50/70 标记且行动不能跨阈值时，`action-graph` 同时扣除 `missingFinalMarkPenalty`。终局轮跨 70 的第 3 个终局标记会获得更高权重，避免 2 标记玩家在 60 多分时继续选择不能立刻补标记的铺垫行动。
- 50/70 门槛的终局板块选择会对当前基础值为 0 的公式扣除 `zeroBaseLatePenalty`，尤其压低第 3 个标记时的空公式首位投机；B1 若三色痕迹最少值仍为 0 且缺色管线不足，会额外扣除 `b1FeasibilityPenalty`，避免 50/70 标记被槽位竞争分推向最终记 0 的痕迹短板公式；C1/C2 若只有 1-2 个已兑现任务且手牌/保留牌里的任务或 3 型牌管线不足，会额外扣除 `weakCFormulaPenalty`，避免低尾玩家用薄任务板抢过更稳的 A/B/D。25 分首标记仍保留早期规划空间。
- B2 终局板块选择会额外检查两侧真实进度：若当前基础值为 0 且自己还没有完成扇区，不能只凭环绕/登陆路线和首槽优先抢 B2；评分会按 `b2FeasibilityPenalty` 扣除无扇区完成或无环绕/登陆一侧的兑现风险，避免高倍率 B2 最终记 0。
- 终局轮仍缺 50/70 标记时，PASS 和主行动后的 `end-turn` 惩罚按下一个缺失门槛的差距动态提高；数据放置候选会显式携带格位/蓝列奖励里的直接得分，避免 28 分/1 标记这类低分玩家因为所有候选为负而直接 PASS，优先尝试任何可继续推进分数的行动。
- 已有 2 个标记且终局轮处于 50-69 分时，AI 会把信用点/手牌/宣传转换成能量或手牌的快速交易纳入候选；若只差少量分数跨 50/70 且交易后能立刻支付扫描成本，也会启用信用点换能量、弃牌换能量或弃牌换信用点来兑现扫描。64 分以上且只缺第 3 个标记时，不再因为 pace deficit 已很小而跳过扫描资源恢复；68 分以上若一信用点短缺会先用宣传精选补牌，再弃牌换信用点，保留扫描后的手牌续航。若当前已经能支付扫描但扫描不能直接跨 70，宣传精选会按“拿到后当前是否可打出、是否能直接跨门槛”重估公共牌，避免理论高价值但短期付不起的牌压过可兑现找分。若 50-61 分、离 70 仍较远、宣传不少于 6、手牌 2-4 且公共牌区存在高价值牌，允许一次宣传精选寻找可打出的得分牌；若宣传不足但能量明显过剩，也允许用能量精选，或在只差少量分数且缺 1 信用点时换信用点准备扫描。第 3 轮以后若分数/终局标记明显落后且手牌、能量或信用点断档，也会有限启用信用点换牌、信用点换能量、手牌足够厚时弃牌换能量，用于给分析、移动、登陆或打牌续资源，避免 50 后资源清空直接 PASS。第 4 轮已有 3 个终局标记但手牌见底、分数仍低于约 150 时，信用点/能量/宣传都可作为精选牌找分来源。若火箭已经在可兑现行星上，且能量交易能把单次直接登陆或单次环绕推近 25/50/70 门槛，信用点/手牌换能量会按行星兑现计划加分；主行动已用完且当前能量已经足够下回合兑现时，会保留这部分能量，不再为了普通信用点恢复把登陆窗口花掉。
- 终局恢复交易继续覆盖低尾：低于 50 且仍缺第 2 个标记时，若信用点归零但能量足够，会考虑 `2能量→1信用点` 来恢复打牌/扫描入口；若宣传足够且公共牌区存在高价值牌，也允许宣传精选找分牌。已有 2 个标记但仍未到 70 时，宣传精选找分牌的价值上调，同时“不直接跨 70 的研究科技”缺标记惩罚提高，避免最后一轮继续铺科技边际而放弃第三标记现金化。终局轮主行动后若尚未 PASS 且仍缺 50/70 标记，后期恢复交易可以提前为下一次轮到自己准备信用点、能量或手牌，不再只限“当前立刻能花掉资源”的窗口。若已经拿满 3 个终局标记但仍低于约 150 分、信用点为 0 且有足够能量/手牌支付扫描，AI 会模拟 `2张牌→1信用点` / `2能量→1信用点` 后的标准扫描，防止 70 多分玩家带着手牌和能量直接 PASS。
- 终局 64-69 分且已有 2 个终局标记时，如果标准扫描能提供直接分但还缺 1 能量，AI 会允许连续的信用点/弃牌换能量来打开扫描找分窗口；避免 66/68 分玩家只做一次半成品资源恢复后下一轮没有 quick 候选只能 PASS。
- 扫描用于判断是否跨 50/70 门槛时只统计固定扇区扫描的可证明直接分；公共牌区扫描和手牌扫描仍按普通收益参与扫描评分，但不再把后续可变目标当成保证直接分，避免误判“扫描可跨 70”后过度交易。
- 星球和卫星奖励里的“收入”不再按固定 `+1信用收入` 估值；AI 会按当前手牌可弃收入角标挑选最优收入牌，多次收入奖励按不同手牌顺序结算预期，避免低估能量/盲抽收入链，也避免没有可弃收入牌时虚高。
- 移动评分会按星球最优步数窗口和三次旋转可带来的距离摆动处理 staging：若移动后仍在窗口外但可通过后续旋转进入窗口，只给有限等待价值；若无橙2进入小行星、或寰宇超动力尝试把多火箭推进小行星且没有即时兑现，则触发寰宇专属卡死惩罚，避免两个火箭一起陷入小行星后空转。移动到非地球行星、彗星和拥有橙2后进入小行星的宣传收益会作为折扣后的到达收益进入移动评分；手牌角标/公司精选角标移动若最佳移动为负收益，不再用移动点基础值包装成正收益。
- 第 3 轮以后若仍低于 25 分且发射已有正收益，标准扫描会被压到发射之后，避免低分玩家继续扫数据而错过建立火箭得分路线；终局已有 2 标记但离 70 仍较远时，若手牌见底且仍有宣传/信用点，允许一次精选牌交易作为最后找分兜底。
- 终局已有 2 标记且基础分在 58-69 时，移动会保护当前可支付扫描的能量窗口；如果移动本身没有直接后续得分且会耗掉可扫描能量，会额外扣分，避免 60 多分玩家在最后一轮先移动后无法扫描补第 3 标记。
- 首痕迹目标已纳入颜色竞争态：隐藏槽位中被别人抢走的颜色不再重复放到 state 额外位拿低收益，AI 会等待揭示后再按物种格子奖励估值；若强制 `alien_trace` 流程已经打开且只剩可放置的 state extra 位，则允许低分 fallback 收口，避免自动批跑卡死。黄色登陆路线会按剩余隐藏首位和当前玩家前序对手的可登陆压力降低可达性，粉色/蓝色目标权重提高。
- 第 3 轮到终局轮持续保留 70 分/三终局标记压力，缺少 25/50/70 标记时 PASS 会被压低；终局轮若还缺标记，无直接得分动作会额外扣分。45-49 分且只有 1 个终局标记时会小幅鼓励直接跨过 50，已到 50 分且拿到 2 个终局标记后，能直接补向 70 分的打牌、环绕、登陆和移动兑现链会获得额外现金化价值；终局缺标记且行动能直接跨过 50/70 时，行动图 net 会再乘小幅缺标记急迫系数。
- 默认策略权重不再全为 1，并按参考行动日志中白色高分路线偏向打牌、科技、终局兑现和任务：`engine=1.1`、`playCard=1.24`、`tech=1.18`、`task=1.08`、`orbitLand=1.12`、`final=1.2`，同时控制长路线和普通移动（`scan=1`、`route=0.94`、`move=0.89`、`pass=0.89`）。AI 初始选择会注入并强制选择 AI 专用公司：每局第一个电脑使用“寰宇超动力”，其余电脑使用“作弊实验室”。层云核心等普通 1x 若正常获得，会按真实能力映射进对应行动风格（层云角标按 `cardCorner`、寰宇移动按 `move`）参与估值；寰宇超动力复用寰宇动力的 1x 双移动/火箭上限，并在每轮开始给 1 能量、1 盲抽、1 宣传，PASS 后追加一次免费发射；作弊实验室复用异星实验室三色板块永久正面，开局获得 3 张盲抽和 4 次收入增加，并在每轮开始给 1 能量和 1 盲抽。移动评分过滤付费且无兑现的中途移动，揭示后外星痕迹按 1-5 号位递增估值并提高外星人牌收益；主行动后的 quick 行动用显式收益修正 action graph，避免正收益放数据、角标或交易被 end-turn 截断。
- 参考行动日志的核心差异不是某张随机牌，而是白色高分玩家持续围绕“手牌供给 -> 打牌/扫描/分析/科技 -> 终局板兑现”滚动：高分样本通常 16-19 次打牌、3-9 个任务、反复分析和科技，后期扫描会服务已标 B2/扇区完成，而不是分散拿普通数据。AI 因此把第 4 轮收入估值修正为 0 次未来 PASS 收入，收入弃牌改为始终动态选择并提高低手牌收入的优先级；C1/C2 终局标记会估算手牌/保留牌里的任务和 3 型牌成长，但 C1 零基础投机会被压低，避免空任务板抢过可兑现 A/B/D。
- 已保留未完成任务现在会反向增强目标路线：星球环绕/登陆任务把实际奖励、直接分、C1/C2 任务增量和低任务压力加到对应星球目标；彗星/小行星等探测器位置任务也会提高对应可见坐标的移动目标值，避免打出任务牌后被普通即时登陆路线完全压过。手牌里的未打出任务牌也会按任务奖励、直接分、C1/C2 增量和低任务压力提高弃置保留成本；若角标移动能立刻衔接登陆、环绕或直接得分，则这项保留成本会打折，避免误杀早期高兑现移动路线。
- 第 4 轮的低分低手牌玩家会把 3 宣传、2 信用点或 2 能量精选视作恢复打牌的 quick 候选：即使已经拿满 3 个终局标记，只要主行动仍开放、分数低于 150 且手牌不超过 1，公共牌评分非负就允许换牌，避免尾盘有资源但没有手牌继续 PASS。
- 终局标记会额外估算 D1/D2 的未来科技成长，避免当前基础值为 0 时低估第一/第二标记；研究科技候选也会把已标和潜在 D1/D2 增量计入价值，并在 D1 面板存在时偏向补最低科技颜色。终局轮已标或可能标 C2 时，3 型终局牌和带终局计分模型的牌会获得打出急迫值，并在角标评分中得到保留惩罚；角标弃牌的手牌压力只在明显溢牌时放大，任务/终局/3 型牌会折减，防止把“大巡游计划”这类牌当资源角标消耗。终局主行动仍可用且信用点归零时，AI 会模拟 `2张牌→1信用点` / `2能量→1信用点` 后的可打牌候选，仅在能立刻解锁高价值打牌、任务、3 型或终局增量时交易，并用交易弃牌保留价值保护被解锁的牌。若已标/可能标 B2 且完成扇区是瓶颈，扫描目标会优先选择能让自己赢下或接近赢下扇区的星云，避免把最后槽位送给对手。效果流里的无目标外星痕迹、失败奖励扫描和公共扫描空目标会按 skipped 自动收口，避免批跑停在 active effect。
- 第 3 轮以后若玩家已经拿到 2-3 个终局标记但任务完成数仍为 0-1，且手牌/保留牌/终局板存在实际 C1/C2 或任务管线，会进入“低引擎尾盘恢复”压力：任务牌、C2 相关 3 型牌、终局计分牌、带科技/扫描/抽牌效果的打牌候选会获得额外 `lateCardEnginePressure`；已标/可能标 D1/D2 但科技数仍低于 8 时，研究科技候选会获得 `lateTechCatchupValue`。这条压力只抬高可兑现引擎行动，不再泛化压低天王星/海王星卫星、登陆/环绕等路线现金化。
- 主行动前的 `2张牌→1信用点` / `2能量→1信用点` 打牌解锁不再只限终局轮：第 2 轮以后若当前信用点为 0，且交易后能立刻打开明显更高价值的打牌候选，AI 会把该交易作为 quick 候选；评分仍扣除被弃牌/资源机会成本，并保护被解锁的高价值牌。终局轮 70-114 分、手牌不少于 3 张且当前没有任何可支付手牌时，允许 `2张牌→1信用点` 补第二个信用点，但仍必须通过同一套高价值打牌模拟，避免把正常路线玩家拉去做泛用资源交易。
- 低尾发射不再只看“当前没有更好主行动”：第 3 轮若仍不高于 42 分、还没第 2 个终局标记，并且发射会花掉最后信用点且没有后续移动预案，AI 会重罚该发射；终局轮缺标记且无后续路线的发射也会降分，避免只因 PASS 惩罚更高而空耗信用点。
- 卡牌左上角移动不再只看抽象路线分：候选会保留移动预览，并在没有登陆/环绕/直接得分跟进时对任务牌、3 型终局牌、外星人牌施加保留惩罚，避免为 1 移动消耗关键牌。`b_49.webp` 引力弹弓的“访问非地球行星后支付 1 宣传获得 1 移动”后续移动在 AI 中禁止再次落到非地球行星；若当前火箭已经停在可环绕/登陆的非地球行星上，也不会为了额外 1 移动离开兑现点，避免火星/金星等相邻行星之间自触发循环；无安全候选时效果会跳过并收口。
- 当前浏览器批跑样本显示四电脑全体均分已越过 180 且上限可达 300：固定种子 `codex-continuation-baseline` 四电脑 4 局在 b1 空公式保护和三标低分扫描恢复后全体分数均值约 `198.75`、最高 `311`、最低均分 `158.25`、赢家均分 `243.5`，`blockedGames=0`、`bugCounts={}`。压力样本 `codex-huanyu-varied-v3` 四电脑 2 局全体分数均值约 `185.63`、最高 `251`、最低均分 `142.5`、赢家均分 `231`，仍作为不同种子的低尾压力参考。
- 批跑样本的低尾诊断保留向后兼容的 `lowMarkPlayerDiagnostics`（最差玩家）并新增 `lowMarkPlayerDiagnosticsList`，覆盖所有未满 3 终局标记或基础分低于 70 的玩家，便于后续按完整低尾集合分析“资源耗尽、只差门槛、路线未兑现”等共性。
- 默认人机入口为 1 人类 + 3 电脑；批跑、A/B 和调参循环未显式传人数时按 4 人局重置，用于放大旋转、扫描、扇区和首痕迹竞争。
- AI 会在手牌超过上限或低可打出价值牌过多时枚举卡牌左上角快速行动，并可自动收口移动角标的免费移动。
- 未揭示槽位的首痕迹按“5 分 + 宣传 + 预期外星人牌/物种揭示收益 + 对手失去首标的竞争差”动态估值；九折按机会/阈值估值，不当作普通发牌外星人。
- 非 debug 外星人痕迹奖励在进入 picker 前必须固化 `targetPlayerId/targetPlayerColor`；AI 普通流程只点击 picker、隐藏首痕迹空位或已揭示牌图/可用 state 位，隐藏阶段被别人抢走的颜色不会继续落到 state 额外位。
- 揭示后的外星人使用子流程不再只停在弹层：九折机会、异常点拿牌/角标、半人马奖励/条件、虫族任务/拿牌/化石、阿米巴拿牌/符号/痕迹移除、奥陌陌拿牌、符文族拿牌/符号分支会进入 AI 子决策；方舟使用弹层会在解锁与放置间按格位奖励和终局阈值兑现估值。
- 符文族揭示后，AI 会把持有 symbol 放入黑圈作为快速行动候选，并按黑圈奖励、已映射符文族牌收益、symbol 终局库存损失和 source symbol 获取价值动态估值；依赖 symbol 奖励/分支的符文族即时打牌候选会等到相关 symbol 已放入黑圈且有可结算奖励后才放开，普通任务/资源符文族牌仍可按自身收益打出；阿米巴揭示后的痕迹格会按对应颜色区域当前 symbol 的实际奖励估值。
- 行动日志新增 `alien-use` 与 `data-placement`，可进入序列挖掘与玩家画像统计。
- 主行动打牌已放开半人马、阿米巴、奥陌陌、符文族的第一批即时效果估值；虫族搬运牌会把任务完成后的化石重复、数据、拿牌、直接分和蓝格解锁计入链条价值，木星/土星路线按未揭示化石期望值或已查看化石最高值、任务奖励、蓝格解锁和后续运输移动成本综合评分，化石搬运棋子会把任务目的地作为自身路线目标；虫族登陆/环绕牌仍可按普通星球收益打到其它合法目标。虫族化石选择也按当前资源/续航收益排序；奥陌陌化石按 4 化石兑现 25 分的路线价值计入痕迹、扫描、星球奖励和打牌估值，2/4 号位与扇区结算会作为化石构筑，1 号位和化石换数据/扫描/移动按真实机会成本扣分；方舟解锁会按当前解锁数、可新开放的 2/3/4 号痕迹位、手中未打出的方舟牌数量和 2 信用点支付能力估值，不再盲目先解锁 3 张；已解锁方舟牌的打牌估值会按当前奖励牌堆的高级奖励期望动态计算。
- `analyze` 与 `placeData` 已进入候选集合，优先服务“6 数据分析”和扇区扫描路线。

## 9. 自博弈与日志挖掘

复用 `runAiAutoBattleBatch` / `runAiStrategyABTest` / `runAiStrategyTuningCycle`，并增强“从高分日志反推可复用路径”：

1. 批量自博弈，按最终分分桶，抽取**高分局（>25 / 抢到首痕迹）**完整序列。
2. `battle-analytics.js` 增加**可配置窗口的行动序列**频繁模式统计，默认统计每名玩家自己的前 6 个玩家回合，支持 `sequenceWindowTurns: 8/10/"all"`；同时支持玩家回合序列、全局行动序列、前 N 个主行动序列。
3. 每条序列 token 包含行动 id、`plan.type`、路线目标、移动后续主行动、终局标记、关键子决策、回合/轮信息；当前已覆盖 `turn-action`、`scan-target`、`land-target`、`alien-trace`、`alien-use`、`data-placement`、`final-score-mark` 等关键决策。
4. 按最终分构成分桶挖掘路径：高总分、高板块分、高任务分、高科技分、高卡牌分、抢到首痕迹、第一轮 25 分。
5. 把高频高分序列固化为 L4 的**开局剧本（opening book）**，开局优先尝试已验证路径。
6. 同 seed A/B 验证；图谱常量 + 目标权重 → 自博弈 → 日志挖掘 → A/B → 回灌，形成可解释闭环。不得只凭“胜者相关性”直接写死策略。

每个候选的 `breakdown`（gain/cost/finalMarginal/goalBonus/feasibility）入日志，使调参从“猜权重”变为“看明细”。
